GeoSPARQL 1.0 Conformance Class: Feature Classcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/core/feature-class
Definition

Implementations shall support the SPARQL Query Language for RDF, the SPARQL Protocol for RDF and the SPARQL Query Results XML Format.

GeoSPARQL 1.0 Conformance Class: SPARQL Protocolcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/core/sparql-protocol
Definition

Implementations shall support the SPARQL Query Language for RDF [SPARQL], the SPARQL Protocol [SPARQLPROT] and the SPARQL Query Results XML [SPARQLRESX] and JSON [SPARQLRESJ] Formats.

GeoSPARQL 1.0 Conformance Class: Spatial Object Classcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/core/spatial-object-class
Definition

Implementations shall support the SPARQL Query Language for RDF, the SPARQL Protocol for RDF and the SPARQL Query Results XML Format.

GeoSPARQL 1.0 Conformance Class: Feature Propertiescon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/feature-properties
Definition

Implementations shall allow the properties geo:hasGeometry and geo:hasDefaultGeometry to be used in SPARQL graph patterns.

GeoSPARQL 1.0 Conformance Class: Geometry asGML Literalcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/geometry-as-gml-literal
Definition

Implementations shall allow the RDF property geo:asGML to be used in SPARQL graph patterns.

GeoSPARQL 1.0 Conformance Class: Geometry asWKT Literalcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/geometry-as-wkt-literal
Definition

Implementations shall allow the RDF property geo:asWKT to be used in SPARQL graph patterns.

GeoSPARQL 1.0 Conformance Class: Geometry Classcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/geometry-class
Definition

Implementations shall allow the RDFS class geo:Geometry to be used in SPARQL graph patterns.

GeoSPARQL 1.0 Conformance Class: Geometry Propertiescon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/geometry-properties
Definition

Implementations shall allow the properties geo:dimension, geo:coordinateDimension, geo:spatialDimension, geo:isEmpty, geo:isSimple, geo:hasSerialization to be used in SPARQL graph patterns.

GeoSPARQL 1.0 Conformance Class: GML Literalcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/gml-literal
Definition

All geo:gmlLiterals shall consist of a valid element from the GML schema that implements a subtype of GM_Object as defined in [OGC 07-036].

GeoSPARQL 1.0 Conformance Class: GML Literal Emptycon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/gml-literal-empty
Definition

An empty geo:gmlLiteral shall be interpreted as an empty geometry.

GeoSPARQL 1.0 Conformance Class: GML Profilecon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/gml-profile
Definition

Implementations shall document supported GML profiles.

GeoSPARQL 1.0 Conformance Class: Query Functionscon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/query-functions
Definition

Implementations shall support geof:distance, geof:buffer, geof:convexHull, geof:intersection, geof:union, geof:difference, geof:symDifference, geof:envelope and geof:boundary as SPARQL extension functions, consistent with the definitions of the corresponding functions (distance, buffer, convexHull, intersection, difference, symDifference, envelope and boundary respectively) in Simple Features [ISO 19125-1].

GeoSPARQL 1.0 Conformance Class: SRID Functioncon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/srid-function
Definition

Implementations shall support geof:getSRID as a SPARQL extension function.

GeoSPARQL 1.0 Conformance Class: WKT Axis Ordercon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/wkt-axis-order
Definition

Coordinate tuples within geo:wktLiterals shall be interpreted using the axis order defined in the spatial reference system used.

GeoSPARQL 1.0 Conformance Class: WKT Literalcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/wkt-literal
Definition

All RDFS Literals of type geo:wktLiteral shall consist of an optional URI identifying the coordinate reference system followed by Simple Features Well Known Text (WKT) describing a geometric value. Valid geo:wktLiterals are formed by concatenating a valid, absolute URI as defined in [RFC 2396], one or more spaces U+0020 character) as a separator, and a WKT string as defined in Simple Features [ISO 19125-1].

GeoSPARQL 1.0 Conformance Class: WKT Literal Default SRScon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/wkt-literal-default-srs
Definition

The URI http://www.opengis.net/def/crs/OGC/1.3/CRS84 shall be assumed as the spatial reference system for geo:wktLiterals that do not specify an explicit spatial reference system URI.

GeoSPARQL 1.0 Conformance Class: WKT Literal Emptycon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-extension/wkt-literal-empty
Definition

An empty RDFS Literal of type geo:wktLiteral shall be interpreted as an empty geometry.

GeoSPARQL 1.0 Conformance Class: BGP RDFS Entailmentcon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-topology-extension/bgp-rdfs-ent
Definition

Basic graph pattern matching shall use the semantics defined by the RDFS Entailment Regime [W3C SPARQL Entailment].

GeoSPARQL 1.0 Conformance Class: EH Query Functionscon

URI http://www.opengis.net/spec/geosparql/1.0/conf/geometry-topology-extension/eh-query-functions
Definition
